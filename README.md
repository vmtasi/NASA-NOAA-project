The system categorizes Kepler Objects of Interest (KOIs) into three distinct classes:\n1.  **CONFIRMED:** Validated Exoplanets.\n2.  **CANDIDATE:** Potential exoplanets requiring further verification.\n3.  **FALSE POSITIVE:** Likely eclipsing binaries or instrumental noise.\n\nThis project focuses on a domain-driven approach, prioritizing physical transit characteristics over meta-data to ensure the model learns astrophysical properties rather than pipeline artifacts.\n\n## üìä Dataset\n**Source:** [NASA Exoplanet Archive - Kepler Objects of Interest (KOI) Table](https://exoplanetarchive.ipac.caltech.edu/cgi-bin/TblView/nph-tblView?app=ExoTbls&config=cumulative)\n\n### Key Features Used\nTo prevent data leakage, we specifically selected physical features derived from light curve analysis:\n*   `koi_period`: Orbital Period (days).\n*   `koi_duration`: Transit Duration (hours).\n*   `koi_depth`: Transit Depth (parts per million).\n*   `koi_impact`: Impact Parameter.\n*   `koi_steff`, `koi_slogg`, `koi_srad`: Stellar properties (Temperature, Gravity, Radius).\n\n## üõ†Ô∏è Methodology\n\n### 1. Feature Engineering & Preprocessing\n*   **Leakage Prevention:** Removed meta-scores (e.g., `koi_score`) generated by the Kepler pipeline to ensure the model learns from physical data.\n*   **Missing Data Strategy:** Implemented a \"Missingness as a Feature\" strategy by creating binary flags for missing rows before imputing values with the median.\n*   **Log-Normal Transformation:** Applied `log1p` transformations to right-skewed features (`period`, `duration`, `depth`) to normalize distributions and handle astronomical outliers.\n*   **Scaling:** Applied Standard Scaling to normalize feature ranges for Neural Network stability.\n\n### 2. Model Architecture\nWe compared three distinct modeling approaches:\n1.  **Logistic Regression:** A linear baseline to establish lower-bound performance.\n2.  **Random Forest Classifier:** An ensemble method to capture non-linear relationships and feature interactions.\n3.  **Multi-Layer Perceptron (MLP):** A Deep Neural Network (TensorFlow/Keras) with Dropout layers for regularization.\n\n## üìà Results\nThe dataset is imbalanced (approx. 50% False Positives). We optimized for **F1-Score** and **Precision** to ensure high confidence in Confirmed planets.\n\n| Model | Accuracy | Macro F1-Score | Key Insight |\n| :--- | :--- | :--- | :--- |\n| **Logistic Regression** | 59% | 0.57 | Fails to capture complex boundaries. |\n| **Random Forest** | 73% | 0.66 | Strong performance, good interpretability. |\n| **MLP (Neural Net)** | **74-75%** | **0.68** | **Best Performer.** Superior precision on False Positives. |\n\n**Final Selection:** The **MLP model** was selected for production due to its superior ability to distinguish between `CANDIDATE` and `CONFIRMED` classes.\n\n## üíª Tech Stack\n*   **Language:** Python 3.8+\n*   **Data Manipulation:** Pandas, NumPy\n*   **Visualization:** Matplotlib, Seaborn\n*   **Machine Learning:** Scikit-Learn (Logistic Regression, Random Forest, Preprocessing)\n*   **Deep Learning:** TensorFlow / Keras\n\n## ‚öôÔ∏è Installation & Usage\n\n1.  **Clone the repository:**\n    ```bash\n    git clone https://github.com/your-username/exofinder.git\n    cd exofinder\n    ```\n\n2.  **Install dependencies:**\n    ```bash\n    pip install pandas numpy matplotlib seaborn scikit-learn tensorflow\n    ```\n\n3.  **Run the Notebook:**\n    Open `ExoFinder.ipynb` in Jupyter or Google Colab to see the full training pipeline.\n\n4.  **Inference (Example):**\n    The project includes a helper function `predict_disposition` for real-time predictions:\n    ```python\n    #
